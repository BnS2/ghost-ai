# Canvas Ergonomics Patch: Selection and Deletion

Add explicit selection and deletion behavior for canvas elements as a focused patch on top of `17-canvas-ergonomics.md`.

## Implementation

1. Support keyboard deletion.
   - selected nodes and edges should be removable with `Backspace`
   - selected nodes and edges should also be removable with `Delete`
   - deletion shortcuts should not run while users are typing in editable fields

2. Add a visible selected-node delete action.
   - selected nodes should show a compact trash action in the existing floating node toolbar
   - keep the action visually subtle and consistent with the dark canvas UI
   - prevent toolbar interactions from dragging nodes or panning the canvas

3. Add visible selected-element deletion.
   - users should be able to select multiple nodes with a click-drag selection box
   - selection boxes should use partial selection so nodes do not need to be perfectly enclosed
   - the bottom canvas controls should show a visible "Delete selected" action only when elements are selected
   - the action should show the selected element count when available

4. Keep deletion connected to the existing collaborative canvas state.
   - use the React Flow deletion path that already integrates with `useLiveblocksFlow`
   - deleting a node should remove connected edges through React Flow's normal element deletion behavior
   - keep undo/redo behavior available through Liveblocks history

## Scope Limits

- don't add confirmation dialogs for canvas element deletion
- don't change shape rendering, edge rendering, or label editing
- don't add server persistence behavior in this unit

## Check When Done

- Selected nodes can be deleted from the node toolbar.
- Multiple nodes can be selected with a click-drag selection box.
- The bottom canvas delete action appears only when elements are selected.
- Selected nodes and edges can be deleted from the visible bottom canvas action.
- Selected nodes and edges can be deleted with `Backspace`.
- Selected nodes and edges can be deleted with `Delete`.
- Text editing fields do not trigger canvas deletion.
- Deletion updates flow through the Liveblocks-backed React Flow state.
- `npm run build` passes.
