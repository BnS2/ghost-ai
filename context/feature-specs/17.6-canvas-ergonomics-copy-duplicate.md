# Canvas Ergonomics Patch: Copy, Paste, and Duplicate

Add familiar copy, paste, and duplicate behavior for reusable canvas shapes as a focused patch on top of `17-canvas-ergonomics.md` and `17.5-canvas-ergonomics-deletion.md`.

## Implementation

1. Support selected-node copy.
   - selected nodes should be copyable with `Cmd/Ctrl + C`
   - selected nodes should also be copyable from a visible node toolbar action
   - copy shortcuts should not run while users are typing in editable fields
   - copied node data should preserve shape, label, color, and dimensions
   - copied node data should not preserve selection state

2. Support selected-node paste.
   - copied nodes should be pasteable with `Cmd/Ctrl + V`
   - pasted nodes should receive new unique IDs
   - pasted nodes should appear offset from the source nodes so the result is immediately visible
   - pasted nodes should become selected after insertion
   - repeated paste actions should keep offsetting from the last pasted position

3. Support selected-node duplicate.
   - selected nodes should be duplicable with `Cmd/Ctrl + D`
   - selected nodes should also be duplicable from a visible node toolbar action
   - duplicate should behave like copy plus paste in one action
   - duplicated nodes should receive new unique IDs, appear offset, and become selected

4. Support multi-node reuse.
   - copying or duplicating multiple selected nodes should preserve their relative positions
   - when all selected nodes are connected only to other selected nodes, duplicate those internal edges too
   - duplicated edges should receive new unique IDs and point to the duplicated node IDs
   - do not duplicate edges that connect to unselected nodes

5. Keep reuse connected to the existing collaborative canvas state.
   - create pasted or duplicated elements through the same React Flow node and edge change path used by the Liveblocks-backed canvas
   - inserted nodes and edges should be undoable through Liveblocks history
   - do not use server persistence or browser storage in this unit

## Scope Limits

- don't add cross-project clipboard support
- don't copy canvas viewport, zoom, or selection box state
- don't copy Liveblocks presence or collaborator state
- don't add server persistence behavior in this unit
- don't change shape rendering, edge rendering, label editing, or deletion behavior

## Check When Done

- Selected nodes can be copied with `Cmd/Ctrl + C`.
- Selected nodes can be pasted with `Cmd/Ctrl + V`.
- Selected nodes can be duplicated with `Cmd/Ctrl + D`.
- Selected nodes can be copied or duplicated from visible toolbar actions.
- Pasted and duplicated nodes preserve shape, label, color, and dimensions.
- Pasted and duplicated nodes receive new IDs and are offset from the source nodes.
- Multi-node copy and duplicate preserve relative positions.
- Internal edges between copied or duplicated selected nodes are recreated with new IDs.
- Edges connected to unselected nodes are not duplicated.
- Shortcut handling skips editable fields.
- Copy, paste, and duplicate updates flow through the Liveblocks-backed React Flow state.
- Undo and redo work for pasted and duplicated nodes through Liveblocks history.
- `npm run build` passes.
